## 2. WBS 分解思路
本项目WBS采用面向交付物的方法进行分解，确保每一项均为可检查、可交付的成果，而非过程性活动。  

分解逻辑如下：  
 一级：项目整体成果（受控配置项）  
二级：按成果类型划分（仓库、WBS文档、进度计划、配置管理、总结报告）  
三级：细化为最小可管理成果单元（如清单、说明、记录等）  

例如：  
WBS文档 → 成果分解表 → L3成果清单  
配置管理 → 冲突处理 → 冲突解决记录  

该分解方式保证了：  
结构完整  
逻辑清晰  
可直接支撑进度计划编制

## 4. 配置管理与协作过程
本项目采用Git进行配置管理，建立了规范的分支与协作机制：  
（1）配置项识别  
    结合项目实际仓库结构，识别主要配置项如下：  
项目说明类：  
    README.md（项目总体说明与使用说明）  
    项目管理文档（docs目录）  
        01-project-charter.md（项目启动说明）  
        02-wbs.md（WBS分解文档）  
        03-schedule.md（进度计划文档）  
        04-config-plan.md（配置管理文档）  
        05-summary-report.md（总结报告）  
        conflict.md（冲突实验文件）  
其他资源：  
    assets（资源文件）  
    src（示例代码或占位）  
    test（测试相关文件）  
  
（2）分支策略  
项目采用基础分支管理模型：  
main分支：最终成果分支，所有最终提交均在该分支完成  
dev分支：阶段集成分支（用于过程集成）  
feature分支：成员个人开发分支（如 feature/wbs、feature/schedule）  
（3）协作流程  
团队协作流程如下：  
成员在各自 feature 分支进行修改  
完成修改后提交代码（commit）  
通过 Pull Request 发起合并请求  
配置管理员审核并执行合并  
必要时执行分支同步（main → feature）  
最终统一合并至 main 分支  

在提交日志中可以观察到如下操作：  
多成员对 conflict.md 文件进行修改  
执行 Merge pull request 完成分支合并  
执行 Merge branch 'main' into feature/schedule 实现分支同步  

（4）提交与合并规范  
为保证项目规范性，团队制定如下规则：  
提交信息应简洁明确（如 Update、Merge）  
提交人名称统一使用拼音格式  
重要修改通过分支进行，不直接在 main 分支修改  
合并操作优先通过 Pull Request 完成  
所有最终成果统一汇总至 main 分支  

（5）配置管理效果  
通过以上配置管理策略，实现了：  
多人协作过程可追溯  
各阶段成果版本清晰  
冲突可控且可复现  
项目结构规范统一  


## 5. 冲突解决过程记录
在本实验中，小组成员基于 conflict.md 文件人为制造冲突场景。  
（1）冲突产生  
两名成员在不同分支对同一行内容进行了修改：  
版本A：直接合并到 main  
版本B：先经过 dev 再合并到 main  

（2）冲突识别  
在分支合并过程中，由于内容不一致，Git无法自动合并，需要人工处理。  
通过提交记录与diff对比，可以确认冲突来源。  

（3）冲突解决  
团队通过讨论确定最终方案：  
采用“直接合并到 main”  
处理步骤如下：  
对比两个版本内容  
删除未采用版本  
保留最终统一版本  
提交修改并完成合并  

## 6. 经验与问题
（1）实验收获  
通过本次实验，小组对WBS成果导向分解方法有了更清晰的认识，能够从交付物出发进行结构化拆分。在此基础上，进一步掌握了从WBS提取关键活动并编制简化进度计划的方法，使任务安排更加合理。同时，通过使用Git进行协作，熟悉了分支开发、合并及版本管理流程，并通过冲突实验理解了版本冲突的产生原因及基本解决方法。  

（2）存在问题  
在实验过程中，主要问题体现在分支使用和协作规范方面，例如个别操作未严格按照分支流程执行，提交说明不够规范，影响了日志的可读性。此外，对冲突的理解仍停留在操作层面，对其产生机制和处理流程掌握还不够深入，多人同时修改同一文件时缺乏沟通，也增加了冲突发生的可能。  

（3）改进建议  
后续可以通过规范分支使用流程、细化提交说明以及控制提交粒度来提高版本管理质量，同时加强分支同步和团队沟通，减少冲突发生。并通过多次练习冲突处理过程，加深对Git协作机制的理解，提高实际操作能力。
