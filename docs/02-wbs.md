# 02 WBS 文档

> 要求：WBS 必须围绕交付物（成果）分解，至少分解到三级。

## 1. 一级交付物
1.管理端交付物：面向管理人员的活动组织与数据处理成果  
2.客户端交付物：面向普通学生的报名与签到交互成果   
3.项目管理成果：确保项目有序进行的各类过程文档与协作留痕

## 2. 二级与三级分解
1. 管理端交付物  
   1.1 活动管理模块设计   
      1.1.1 活动发布表单原型  
      1.1.2 报名名单审核逻辑说明    
   1.2 统计与导出功能   
      1.2.1 签到率统计图表定义  
      1.2.2 活动总结 Excel 导出模板   

2. 客户端交付物  
   2.1 用户报名界面  
      2.1.1 个人信息输入组件  
      2.1.2 报名成功确认页  
   2.2 签到核心组件    
      2.2.1 动态二维码生成逻辑  
      2.2.2 扫码状态反馈界面    

3. 项目管理成果   
   3.1 规划类文档  
      3.1.1 项目 WBS 分解表  
      3.1.2 简化进度计划表  
   3.2 协作与总结类成果   
      3.2.1 冲突解决过程记录   
      3.2.2 实验综合总结报告  

## 3. 分解说明
本组 WBS 分解逻辑如下：
1.坚持结果导向：严格遵循实验要求，所有三级条目均以“原型”、“说明”、“表”、“报告”等名词形式呈现，杜绝了“开会”、“讨论”等纯动作描述，确保每个任务都有可核查的交付物   
2.业务逻辑完整性：围绕项目背景“校园活动报名与签到管理系统”，将核心功能拆分为管理端和客户端，涵盖了从活动发布到最终导出总结的全生命周期成果  
3.角色任务匹配：分解出的L3级单元直接对应项目经理、计划负责人、配置管理员的工作范围，便于后续在进度计划中明确责任人
